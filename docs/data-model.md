# Structural Data Model

This document serves as the single source of truth for the Academy Builder database schema (powered by the Academy Library core database).

---

## 1. Collection: `assets`
Contains the normalized, unique, and versioned asset metadata documents.

| Field Name | Type | Description |
| :--- | :--- | :--- |
| `asset_id` | **String** (Document ID) | Slugified unique asset identifier (e.g. `why-cloudvision`). |
| `name` | **String** | The display title of the asset. |
| `type` | **String** | The asset format type: `video`, `document`, `quiz`, or `lab`. |
| `version` | **Number** | Incremental version number of the asset (default: `1`). |
| `is_latest` | **Boolean** | Indicates if this document is the active version. |
| `attributes` | **Map** | Metadata blob (detailed below). |

### `attributes` Map Structure
| Key | Type | Description |
| :--- | :--- | :--- |
| `duration` | **Number** | Duration in minutes/seconds or estimated study time. |
| `prerequisite` | **String** | Prerequisites for this asset (can contain formulas or text). |
| `difficulty_level` | **Number** | Decimal scoring representing relative difficulty. |
| `skill_tags` | **Array of Strings** | Learning category tags (e.g. `["NetOps", "CV"]`). |
| `last_updated` | **String** | Date of last update (`YYYY-MM-DD`). |
| `comments` | **String** | Notes from content administrators. |
| `topic` | **String** | Associated curriculum topic. |

---

## 2. Collection: `curriculum_map`
Contains flat nodes defining curriculum tracks, lesson plans, and asset order mapping.

| Field Name | Type | Description |
| :--- | :--- | :--- |
| `id` | **String** (Document ID) | Random autogenerated document ID. |
| `track_id` | **String** | Unique ID of the track (e.g. `automation`). |
| `track_name` | **String** | Friendly display name of the track (e.g. `Automation`). |
| `sub_track` | **String** | Name of the chapter / sub-track (e.g. `Automation Fundamentals`). |
| `lesson` | **String** | Name of the lesson block (e.g. `Automation & NetOps Foundation`). |
| `topic` | **String** | Name of the topic parent (e.g. `Network Automation Overview`). |
| `asset_ref` | **Reference** | DocumentReference to the corresponding `assets` document (maintains referential integrity). |
| `version` | **Number** | Incremental version tracker of the track configuration. |
| `is_latest` | **Boolean** | Indicates if this node belongs to the active curriculum version. |
| `sorting` | **Map** | Ingestion sequence indices (detailed below). |

### `sorting` Map Structure
| Key | Type | Description |
| :--- | :--- | :--- |
| `sub_track_number` | **Number** | Sort index of the sub-track chapter. |
| `lesson_number` | **Number** | Sort index of the lesson block. |
| `topic_number` | **Number** | Sort index of the topic. |
| `sub_topic_number` | **Number** | Sort index of the asset node inside its parent topic. |

---

## 3. Collection: `cms_history`
Stores database snapshots for checkpoints and rollbacks.

| Field Name | Type | Description |
| :--- | :--- | :--- |
| `id` | **String** (Document ID) | Checkpoint commit UUID. |
| `timestamp` | **String** | ISO 8601 creation timestamp. |
| `author` | **String** | User or process initiating the checkpoint (e.g., `Gemini AI Ingest`). |
| `description` | **String** | Explanation of what triggered the write. |
| `state` | **Map** | Deep JSON copy containing full collections data (`assets` and `curriculum_map`). |

---

## 4. Collection: `cache_invalidations`
Logs telemetry change events for consumer client SDK synchronization.

| Field Name | Type | Description |
| :--- | :--- | :--- |
| `id` | **String** (Document ID) | Auto-generated ID. |
| `type` | **String** | Event type: `asset_update`, `curriculum_update`, `database_rollback`. |
| `doc_id` | **String** | ID of the updated document. |
| `change_type` | **String** | Change classification: `added`, `modified`, `removed`, or `null`. |
| `timestamp` | **String** | ISO 8601 execution timestamp. |
| `details` | **Map** | Event-specific metadata (e.g., `{ name: "Asset Title" }`). |
